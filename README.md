# 🚀 알고리즘 스터디 풀이 공유 가이드

본 저장소는 팀원들의 알고리즘 문제 풀이를 기록하고 공유하는 공간입니다. 서로의 코드를 통해 배우고 성장하는 것을 목표로 합니다.

---

## 📁 폴더 구조
각자의 성함을 영문 폴더명으로 생성하여 관리합니다. 플랫폼별로 하위 폴더를 나누어 정리합니다.

```text
.
├── jinook (예시: 본인 이름)
│   ├── baekjoon
│   │   └── Q1234.java
│   └── leetcode
│       └── TwoSum.java
├── team_member_A
└── team_member_B
```

## 📝 파일 명명 규칙
문제의 출처와 번호(혹은 이름)를 명확히 알 수 있도록 작성합니다.

* **백준:** `Q[문제번호].java` (예: `Q1000.java`)
* **프로그래머스:** `[문제이름].java` (예: `타겟넘버.java`)
* **기타:** `[문제이름].[확장자]`

## 💬 커밋 메시지 컨벤션
`main` 브랜치 하나만 사용하므로, 커밋 메시지만 보고도 누가 어떤 문제를 풀었는지 알 수 있게 작성합니다.

* **형식:** `[이름] [플랫폼] [문제번호/이름] [상태]`
* **예시:**
  - `✨ [지눅] 백준 1234번 풀이 완료`
  - `♻️ [지눅] 프로그래머스 타겟넘버 코드 리팩토링`

| 이모지 | 태그 | 설명 |
| :--- | :--- | :--- |
| ✨ | Solve | 문제 풀이 완료 |
| ♻️ | Refactor | 코드 개선 및 리팩토링 |
| 📝 | Docs | 문서 수정 (README 등) |
| 🚑️ | Fix | 오답 수정 |

## 🛠 참여 방법 (Workflow)
우리 스터디는 **`main` 브랜치** 하나만 사용하며, 작업 전 반드시 최신 코드를 불러와 충돌을 방지합니다.

1. **로컬 저장소 업데이트**
   ```bash
   git checkout main
   git pull origin main
   ```
2. **본인 폴더에 풀이 파일 추가** (위의 폴더 구조 및 규칙 준수)
3. **변경 사항 반영**
   ```bash
   git add .
   git commit -m "✨ [이름] [플랫폼] [문제] 풀이 완료"
   git push origin main
   ```

## 💡 스터디 에티켓
* **코드 주석:** 복잡한 로직이나 핵심 알고리즘은 주석으로 간단한 설명을 달아주면 팀원들에게 큰 도움이 됩니다.
* **리뷰 및 질문:** 다른 팀원의 폴더 코드를 직접 수정하지 않습니다. 질문은 해당 커밋의 댓글 기능을 활용하거나 스터디 시간에 진행합니다.
* **시간 엄수:** 정해진 스터디 마감 시간 전까지 풀이를 업로드해 주세요.
